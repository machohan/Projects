Introduction

Write a Java program to handle applications for universities admission providing a much better user interface than in the program discussed in the second lecture. This program should work as an applet with the width = 700 pixels and the height = 500 pixels. The applet should run with JDK’s appletviewer. If you use IDE Java environments such as DrJava make sure that at the end your code also works in JDK. If you do not have JDK working on your home PC you can always use the TEL labs PCs.

 

The tasks of this Java program are similar to those of the example discussed in week 2. This program should allow various users:

- to enter information for a number of candidates/students,

- to admit candidates in the universities chosen

- to display the information about all the candidates stored

- to display the information for a certain candidate

Your Task

In this assignment class Student encapsulates the student information relevant for universities admission: student name, name of the program chosen and 3 universities chosen. As shown in the week 2 Java code, class Student should also contain 3 boolean variables used for the admit status.

Class Undergrad extends class Student and encapsulates the highschool average mark.  Class Postgrad extends class Student and encapsulates the undergrad average mark and the type of postgrad degree (Master or PhD).  For classes Student, Undergrad and Postgrad define constructors and suitable methods for this assignment.

Class ApplicationCentre extends JApplet. At the top of the file introduce as a comment your name and student number.  In this applet you have to create the following Graphics User Interface:

The Main Screen has on the left hand side 4 buttons presented in a column. These buttons have the labels: Input, Admit, DisplayAll, DisplayOne. The user will click on these buttons to change the panels displayed in the right hand side of the screen (using the CardDeck layout manager). The followings are the description of the right panels (the "cards"). Note that you have flexibility in organizing the layout of each panel, but a small percentage of the assignment mark will be associated with creating nice looking panels.

The Input panel, uses labels to prompt the user and four textfields to enter the student name, the name of the chosen program, the average mark and the type of postgrad degree. This panel also contains a JList object containing the following 10 Canadian universities: Toronto, York, Western, Brock, Guelph, Waterloo, McGill, Concordia, Laval and Macmaster. From that list the user will select 3 universities. A Button labeled "Submit" displayed at the bottom of the panel allows the user to enter the input data coming from textfields and JList object into an array with maximum 100 Student objects. Do not forget to erase the content of the textfields when the Submit button is clicked and provide a label which shows how many students were entered (for instance it should say "student 5 out of 100").

      Hint: the 4th textfield content can be used to decide if the Student is Undergrad or Postgrad.  If the textfield is left empty it will mean that one inputs an Undergrad.

The Admit panel is used for the acceptance of applications for Undergrad applicants. At the top this panel has a JComboBox which presents all student names entered in the database (Hint: the creation of the JComboBox requires an array of strings as parameter. This array has to be created during the input process). When the user selects a name from the JComboBox, this program will get the corresponding object from the array of objects. The panel will present three labels with the 3 universities stored in this object, and next to each university two radio buttons (one for accepted, one for rejected). Initially all rejected buttons are checked, but the user will click some accepted radio buttons.

       Hint: You need a different ButtonGroup object for each university.

A "Submit" button at the bottom of this panel will trigger the method setAcceptance() which will change the content of the Student objects. The admission process can then be continued with the selection of a new student name. After the “Submit button is clicked and a new student name is selected all rejected radio buttons should be on.

 

The Admit for Postgrad applicants  is done based on the undergrad average mark.  For all universities the candidates are accepted into a Master program if the average mark is higher or equal to 70% and into a PhD program if the average mark is higher or equal to 80%. 

The DisplayAll panel will present the content of the array of objects using a JTextArea object. For each student (Undergrad or Postgrad) the information comes from the method toString(). The format of the information should be as in the following examples:                                                                                                                                                                                                                                                                                                                                                                                                                        John Smith, average=70, ITEC, Undergrad, York-admitted, McGill-rejected, UofT- admitted.                                                                                                                                                                                                                                                                  Mary Jones, average=90, History, PhD, York-admitted, UofT-admitted, Brock-admitted                                                                                                                                                                                                                                                                                                                                                                                                                                      

 In this textarea the students are displayed with the averages in descending order (you need to sort the array of objects using the Bubble Sort algorithm). The text in the JTextArea is displayed with a Serif style font in Italic and 12-point size.

 The DisplayOne panel  contains at the top the label “Enter student’s name” and next to that label a textfield.  Below the panel contains a JTextArea, where the result of the search is displayed with in Arial style font and 14-point size. If the search for the student fails the textarea should contain the message “Student not found”.
